## 0. 快速摘要（给忙的人看）

- 产品一句话：给个人用户的极简记账工具，用手机快速记账，用 Web 深度查账和导入导出。
- MVP 核心任务：30 秒内完成一笔记账，并能在任意端查到、统计并导出。
- MVP 页面（1-3 个）：移动端「记一笔」/ Web「账单明细」/ Web「报表分析」。
- 设备优先级：手机优先（记账主场景），电脑增强（查账/报表/导入导出）。
- 技术栈：`html-tailwind`（Web） + `React Native`（移动）。
- 风格关键词（3-5 个）：克制、清爽、可信、低负担、高效率。
- 参考链接/截图（可选）：[待补充]
- 本文覆盖范围：导航与布局 / 页面模板 / 组件规范 / 交互与状态 / 文案 / 视觉规范（颜色/字体/间距）。

## 0.1 使用方式（给后续 AI/开发看的）

- 这份文档是 UI 的唯一真相，颜色/字号/间距/组件都以本文为准。
- Web 与移动端共用同一套 tokens（设计变量：全站统一的颜色、字号、间距等），不要各自发明样式。
- 新增页面时先复用第 5 节组件，不够用再补组件规范。
- 任一页面都必须覆盖：加载中、没数据、出错、成功四种状态。

## 1. 范围与原则

### 1.1 本期做什么

- 登录与账号绑定（手机号、微信、邮箱）。
- 移动端快速记账（表单 + AI 对话记账）。
- 分类管理（预置分类 + 自定义分类）。
- 报表分析（分类占比、月度收支趋势、按账户统计、时间段对比）。
- Web 账单筛选与导入导出（CSV、Excel）。

### 1.2 本期不做什么（防止范围膨胀）

- 不做多人协作账本和家庭共享权限。
- 不做投资、基金、股票等资产管理模块。
- 不做发票报销流程与企业审批。
- 不做复杂主题系统，暗黑模式延后。

### 1.3 设计原则（3-5 条，必须可执行）

1. 每页只允许一个主操作按钮，避免用户犹豫。
2. 记账流程优先默认值，能自动填就不让用户手填。
3. 关键动作都要可撤销或可修改，防止误操作污染账本。
4. 图表先保证可读，再谈视觉；不做信息噪音。
5. 反馈必须立即可见：提交后 300ms 内给出状态提示。

### 1.4 一致性铁律（强制）

- Token 优先：颜色、字体、间距、圆角、阴影、动效必须来自 tokens。
- 组件优先：页面只允许用组件组合，不允许临时造样式。
- 写清禁区：每个关键页面至少写 1 条“不要怎么做”。

## 2. 页面结构与导航

### 2.1 导航模型（必选其一）

- 混合模型：移动端底部 Tab + Web 左侧导航。
- 原因：移动端高频任务是“快速记账”，Web 高频任务是“筛选、分析、导入导出”。

### 2.3 全局布局骨架（文字版线框）

- 移动端：
  - 顶部栏：页面标题 + 次要操作（筛选、帮助）。
  - 内容区：卡片或列表，默认单列。
  - 底部栏：4 个 Tab（首页、记一笔、报表、我的）。
  - 全局主操作：`+ 记一笔` 悬浮按钮（首页可见）。
- Web 端：
  - 顶部栏：页面标题 + 主操作按钮 + 用户菜单。
  - 左侧栏：一级导航 6-7 项，当前项高亮。
  - 内容区：筛选工具栏 + 主内容（表格/图表/表单）。
  - 面包屑：仅二级页面显示（如分类编辑页）。

### 2.4 布局规则（让页面长得像同一个产品）

- 页面宽度：
  - Web：最大宽度 `1280px`，居中布局。
  - 移动：以 `360px` 设计基准，适配 `320-430px`。
- 页面内边距：
  - Web：`--space-6`（24px）。
  - 移动：`--space-4`（16px）。
- 区块间距：区块间 `--space-6`，区块内 `--space-3`。
- 页面头部：标题左对齐，主按钮右对齐，辅助操作放主按钮左侧。
- 信息密度：
  - 移动端偏舒展，减少误触。
  - Web 端偏紧凑，提升查账效率。
- 对齐原则：标题、筛选栏、列表列头统一左对齐；数值列统一右对齐。

## 3. 核心流程与关键状态

### 3.1 核心流程（至少 1 条端到端）

**流程 1：移动端快速记一笔（表单）**
1. 进入首页点击 `+ 记一笔`。
2. 输入金额，选择收/支、分类、账户，填写备注。
3. 时间默认当前，可按需修改；标签可选。
4. 点击 `保存账单`。
5. 顶部 Toast 提示“已记账”，并回到首页最新列表。

**流程 2：AI 对话记账**
1. 在记一笔页切换到 AI 输入。
2. 输入“今天午饭35”。
3. 系统解析并生成确认卡片。
4. 用户确认或修改后入账。
5. 若改了分类，系统记录偏好用于下次推荐。

**流程 3：Web 查账并导出**
1. 进入 Web 账单明细页。
2. 设置时间范围、分类、账户筛选条件。
3. 核对结果后点击导出。
4. 选择 CSV 或 Excel 下载。

### 3.2 关键状态（所有页面一致的规则）

- 加载中：
  - 列表和表格使用骨架屏。
  - 按钮加载时在按钮内显示小转圈并禁用重复点击。
- 没数据：
  - 文案：`还没有账单，先记一笔试试。`
  - 行动：提供 `去记一笔` 或 `导入账单` 按钮。
- 出错：
  - 文案模板：`处理失败，请检查网络后重试。`
  - 保留用户输入，不清空已填数据。
- 成功：
  - 使用 Toast：`已保存` / `导入完成` / `导出已开始`。
  - 给下一步操作入口（继续记账、查看报表）。

## 4. 设计系统（Tokens：唯一真相）

### 4.1 风格方向（锁定审美边界）

- 风格描述：整体清爽、专业、可信，突出效率，不做花哨装饰。
- 关键词：克制、清晰、稳定、轻量。
- 不要的风格：重渐变、玻璃拟态、过度阴影、卡通化图标。
- 组件气质：中等圆角，轻边框，轻阴影，信息密度中等偏高。

### 4.2 颜色 tokens（语义优先）

#### 4.2.1 语义色板（Light/Dark）

| Token | Light | Dark | 用在哪里 | 禁止用在哪里 |
|---|---|---|---|---|
| `--color-brand` | `#2563EB` | `#60A5FA` | 主按钮、关键链接、焦点环 | 大段正文 |
| `--color-bg` | `#F8FAFC` | `#020617` | 页面背景 | 卡片主体 |
| `--color-surface` | `#FFFFFF` | `#0F172A` | 卡片、表单面板 | 页面底色 |
| `--color-text` | `#0F172A` | `#F8FAFC` | 标题、正文 | 禁用文案 |
| `--color-muted` | `#475569` | `#94A3B8` | 辅助文案、次要信息 | 主标题 |
| `--color-border` | `#E2E8F0` | `#1E293B` | 分割线、输入框边框 | 主按钮底色 |
| `--color-success` | `#16A34A` | `#22C55E` | 成功提示、正向金额 | 主 CTA |
| `--color-warn` | `#D97706` | `#F59E0B` | 预警提示 | 主 CTA |
| `--color-danger` | `#DC2626` | `#F87171` | 删除、失败提示 | 常规正文 |

#### 4.2.2 交互状态色（必须统一）

- Hover：主按钮改为 `#1D4ED8`；次按钮背景加 `--color-bg`。
- Active：主按钮改为 `#1E40AF`；点击态不缩放。
- Focus：统一 `2px` 外发光环，颜色 `--color-brand`，必须可见。
- Disabled：透明度 `0.5`，同时禁用点击；文字对比度保持可读。

### 4.3 字体与排版 tokens

- 字体：`Inter, PingFang SC, Noto Sans SC, system-ui, sans-serif`。
- 字重策略：标题 `600`，正文 `400`，数字金额可用 `600`。

| Token | 字号 | 行高 | 字重 | 用在什么地方 |
|---|---:|---:|---:|---|
| `--text-h1` | 28 | 36 | 600 | 页面主标题 |
| `--text-h2` | 22 | 30 | 600 | 区块标题 |
| `--text-body` | 14 | 22 | 400 | 正文、表格内容 |
| `--text-small` | 12 | 18 | 400 | 辅助说明、提示 |
| `--text-amount` | 24 | 32 | 600 | 金额输入和总额 |

### 4.4 间距/栅格/布局 tokens

- 内容宽度：Web 最大 `1280px`；移动全宽减去左右安全边距。
- 栅格：Web 使用 `12` 列；列间距 `24px`。
- 响应式断点：`360 / 768 / 1024 / 1280`。

| Token | px | 用在什么地方 |
|---|---:|---|
| `--space-1` | 4 | 最小内边距 |
| `--space-2` | 8 | 控件间距 |
| `--space-3` | 12 | 卡片内部间距 |
| `--space-4` | 16 | 页面移动端边距 |
| `--space-6` | 24 | Web 页面边距、区块间距 |
| `--space-8` | 32 | 页面大分区间距 |

### 4.5 圆角/阴影/边框 tokens

| Token | 值 | 用在什么地方 |
|---|---|---|
| `--radius-sm` | `8px` | 输入框、小按钮 |
| `--radius-md` | `12px` | 卡片、列表容器 |
| `--radius-lg` | `16px` | 弹窗、抽屉 |

| Token | 值 | 用在什么地方 |
|---|---|---|
| `--shadow-sm` | `0 1px 2px rgba(15,23,42,0.06)` | 卡片 |
| `--shadow-md` | `0 10px 30px rgba(15,23,42,0.12)` | 弹窗 |

### 4.6 动效 tokens（少而稳）

| Token | 值 | 用在什么地方 |
|---|---|---|
| `--motion-fast` | `150ms` | Hover 反馈 |
| `--motion-base` | `200ms` | 常规过渡 |
| `--motion-slow` | `300ms` | 抽屉、弹窗 |
| `--ease-standard` | `cubic-bezier(0.2, 0, 0, 1)` | 默认缓动 |

规则：
- 不用大幅缩放和位移动效，防止界面抖动。
- 动效只表示状态变化，不做装饰动画。

### 4.7 图标/插画/图片规范

- 图标库：`Lucide`，统一线宽。
- 默认尺寸：`16 / 20 / 24`。
- 不要：emoji 代替图标、混用实心与线框图标。

## 5. 组件规范（变体 + 状态）

**工具栏（筛选/搜索/次要操作）**

### 5.1 组件清单（先列出来）

| 组件 | 变体（同一组件不同样式） | 尺寸（大小） | 状态（比如禁用/加载中） | 备注 |
|---|---|---|---|---|
| Button | primary/secondary/ghost/danger | sm/md/lg | 默认/悬浮/禁用/加载中 | 仅 1 个主按钮 |
| Input | default/amount/search | md/lg | 默认/聚焦/错误/禁用 | amount 用于金额 |
| Select | default/filter | md | 默认/展开/禁用 | 分类、账户筛选 |
| Tab | underline/pill | md | 默认/激活/禁用 | 移动底部和内容切换 |
| Card | default/summary/chart | md | 默认/悬浮 | 首页和报表 |
| Table | default | md | 加载/空/错误 | Web 账单明细 |
| Dialog | confirm/form | md/lg | 打开/关闭/提交中 | AI 确认卡片、导入确认 |
| Toast | success/error/info | md | 显示/消失 | 全局反馈 |

### 5.2 Button

- primary：提交、保存、确认等主动作。
- secondary：取消、返回、次要操作。
- ghost：工具栏轻操作。
- danger：删除、清空等风险动作。
- 禁区：同一区域出现两个 primary。
- 文案规则：动词开头，最多 6 个字，如“保存账单”“确认入账”。

状态：
- Disabled：不可点击，opacity 50%，光标禁用。
- Loading：显示小转圈，文案改为“保存中…/导入中…”。

### 5.3 表单与校验（全站一致）

- Label 在输入框上方，错误提示在输入框下方。
- 错误文案模板：
  - `请填写金额`
  - `请选择分类`
  - `备注不能为空`
- 校验时机：失焦时轻校验，提交时强校验。
- AI 解析字段校验：解析缺字段时直接高亮缺失项，不允许直接入账。

### 5.4 列表/表格（Web）

- 默认字段：时间、类型、金额、分类、账户、备注、标签、操作。
- 必须支持：筛选、排序、分页。
- 空状态：展示引导按钮“去记一笔”或“导入账单”。
- 选中行高亮规则：背景 `bg-bg`，左侧 `2px` brand 标记。

### 5.5 移动端专用组件

- 底部 Tab Bar：固定高度 `56px`，图标 + 文案。
- FAB（悬浮记账按钮）：圆形高亮，固定右下角，点击直接进记账。
- AI 确认卡片：显示“解析结果 + 可编辑字段 + 三按钮（确认/修改/取消）”。

## 6. 页面规格（文字版线框：开发能照着做）

### 6.0 页面模板（先选模板，再填具体页面）

- 模板 A：列表 + 详情（用于 Web 账单明细、分类管理、账户管理）。
- 模板 B：表单为主（用于移动记一笔、Web 导入）。
- 模板 C：图表看板（用于 Web 报表分析、移动报表）。

### 6.1 页面：移动-记一笔

- 目的：最短路径完成记账。
- 入口：首页 FAB、底部 Tab。
- 主操作：`保存账单`。
- 页面骨架：
  - 顶部：标题“记一笔” + 历史按钮。
  - 内容：金额输入、收支切换、分类、时间、账户、备注、标签。
  - AI 区：输入框 + “AI 解析”按钮。
  - 底部：保存按钮固定。
- 区块顺序：金额 > 类型 > 分类 > 时间 > 账户 > 备注 > 标签 > 提交。
- 字段与文案：
  - 金额：示例 `35.00`（校验：>0）
  - 备注：示例 `午饭`（校验：必填，最多 30 字）
- 交互规则：
  - 保存成功后返回首页并高亮新账单。
  - AI 解析后必须先确认卡片，不能绕过确认直接入账。
- 状态与反馈：
  - 加载中：按钮 loading。
  - 没数据：不适用。
  - 出错：页内红色提示条，保留输入。
  - 成功：Toast `已保存`。
- 响应式：
  - 手机：单列满宽。
  - 电脑：此页不提供。
- 不要（禁区）：不要把“保存”和“AI 解析”都做成主按钮。

### 6.2 页面：Web-账单明细

- 目的：快速定位账单并做筛选。
- 入口：左侧导航。
- 主操作：`导出账单`。
- 页面骨架：
  - 顶部：标题 + 导出按钮。
  - 工具栏：时间、分类、账户、收/支筛选 + 搜索框 + 重置。
  - 内容区：账单表格 + 分页。
  - 详情：点击行打开右侧抽屉查看详情。
- 布局配方（Tailwind）：
```txt
max-w-7xl mx-auto px-6 py-6 space-y-4
flex items-center justify-between
rounded-md border border-border bg-surface shadow-sm
```
- 字段与文案：
  - 搜索占位：`搜索备注或标签`
  - 时间筛选默认：`本月`
- 交互规则：
  - 筛选条件变化后 300ms 防抖刷新。
  - 列头支持金额、时间排序。
- 状态与反馈：
  - 加载中：表格骨架 8 行。
  - 没数据：`没有符合条件的账单` + `重置筛选`。
  - 出错：`加载失败，请重试`。
  - 成功：导出后 Toast `导出任务已开始`。
- 响应式：
  - 电脑：完整表格。
  - 平板：隐藏低优先字段（标签、操作）。
- 不要（禁区）：不要默认展开高级筛选，占空间且干扰主任务。

### 6.3 页面：Web-报表分析

- 目的：查看收支结构和趋势。
- 入口：左侧导航。
- 主操作：`导出报表`。
- 页面骨架：
  - 顶部：标题 + 时间范围选择。
  - 上区：分类占比、月度收支趋势。
  - 下区：按账户统计、时间段对比。
- 区块顺序：筛选 > 关键指标卡 > 4 图表。
- 字段与文案：
  - 指标卡：总支出、总收入、结余、同比。
- 交互规则：
  - 切换时间范围时所有图表联动刷新。
  - 图例点击可隐藏某分类/账户。
- 状态与反馈：
  - 加载中：图表骨架和占位线。
  - 没数据：`当前时间段暂无账单` + `去记一笔`。
  - 出错：`报表生成失败，请重试`。
  - 成功：筛选刷新无弹窗，使用轻提示。
- 响应式：
  - 电脑：2x2 网格。
  - 手机：纵向单列卡片。
- 不要（禁区）：不要在同一屏超过 4 张图，避免认知过载。

### 6.4 页面：Web-导入导出

- 目的：导入历史账单并导出备份。

## 7. 文案与语气（让产品说人话）

- 整体语气：专业、简洁，不装腔，不喊口号。
- 常用按钮文案：`保存账单`、`确认入账`、`导出账单`、`开始导入`、`重试`。
- Toast 模板：
  - 成功：`已保存`、`导入完成`。
  - 失败：`保存失败，请重试`、`导入失败，请检查文件格式`。
- AI 解析提示：
  - 成功：`已解析，请确认后入账`。
  - 不完整：`缺少账户信息，请补全后保存`。

## 8. 无障碍与可用性（最低要求）

- 键盘操作：Web 所有按钮、筛选、弹窗都可 Tab 到并回车触发。
- 焦点样式：必须显示 `focus ring`，不能只靠颜色变化。
- 对比度：正文和背景对比不低于可读标准。
- 点击热区：移动端可点击区域最小 `44x44px`。
- 图表可读性：颜色区分之外，辅以图例、标签或纹理。
