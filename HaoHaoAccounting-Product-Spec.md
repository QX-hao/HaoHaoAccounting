## 产品说明

HaoHaoAccounting 是一款面向个人用户（上班族、学生党）的极简记账工具，核心目标是让用户在手机端用最低操作成本完成记账，并在 Web 端完成查账、报表分析和数据导入导出。产品通过“强制登录 + 多端实时同步”保证数据一致性，并通过 AI 对话式记账与 AI 自动分类降低手动输入成本，解决“记账麻烦、坚持不下去、回顾困难”的问题。

## 功能需求

- 支持登录与账号体系：提供手机号、微信、邮箱登录，并要求登录后使用
- 支持多端数据同步：手机端与 Web 端账本实时同步，保障同一账号数据一致
- 支持快速记一笔：在手机端快速录入收支，最少必要操作完成入账
- 支持分类管理：提供预置分类并允许用户自定义分类
- 支持账单查询：按时间、分类、账户进行筛选和检索
- 支持账单报表：展示分类占比、月度收支趋势、按账户统计、时间段对比
- 支持导入账单：支持 CSV、Excel 格式导入
- 支持导出账单：支持 CSV、Excel 格式导出
- 支持 AI 对话式记账：解析自然语言（如“今天午饭35”）并生成待确认账单
- 支持 AI 自动分类：根据金额描述、历史习惯自动推荐分类，并持续学习用户修正

## UI 布局

### 整体布局

- 手机端（React Native）：底部四 Tab 布局，`首页`、`记一笔`、`报表`、`我的`
- Web 端（React + Next.js）：左侧导航 + 右侧主内容两栏布局，比例约 `20% : 80%`
- 固定元素：
  - 手机端底部 Tab 始终可见
  - Web 端左侧导航固定，右侧内容区独立滚动

### 手机端布局与区域内容

#### 1) 首页（账本概览）

- 顶部：当月收支摘要卡片（收入、支出、结余）
- 中部：最近账单列表（按时间倒序）
- 底部悬浮主按钮：`+ 记一笔`（主色高亮，右下角固定）

#### 2) 记一笔（核心页面）

- 录入方式切换：
  - 默认 `表单记账`
  - 次入口 `AI 对话记账`
- 表单记账控件规范：
  - 金额输入框：数字键盘，必填，页面顶部，字号最大
  - 收/支切换：分段控件，必填，默认 `支出`
  - 分类下拉：必填，默认根据收/支给出最近常用分类
  - 时间选择：默认当前时间，可手动调整
  - 账户选择：必填，单选，下拉或弹层选择
  - 备注输入：必填，单行输入框
  - 标签选择：选填，可多选
  - 提交按钮：`保存账单`，页面底部固定按钮
- AI 对话记账控件规范：
  - 对话输入框：支持自然语言（示例占位符“今天午饭35”）
  - 解析后确认卡片：展示金额、收/支、分类、时间、账户、备注
  - 操作按钮：`确认入账`、`修改后入账`、`取消`

#### 3) 报表

- 顶部：时间范围筛选（本月/上月/近3个月/自定义）
- 中部图表：分类占比（饼图）+ 月度收支趋势（折线/柱状）
- 底部图表：按账户统计 + 时间段对比

#### 4) 我的

- 登录方式绑定管理（手机号/微信/邮箱）
- 账户管理（现金、银行卡、支付宝、微信）
- 导入导出入口
- 分类管理入口

### Web 端布局与区域内容

#### 1) 左侧导航（20%）

- 菜单项：`总览`、`账单明细`、`报表分析`、`导入导出`、`分类管理`、`账户管理`、`个人设置`
- 交互：当前菜单高亮，点击切换右侧内容区

#### 2) 右侧主内容区（80%）

- 总览页：与手机端首页一致的收支摘要 + 最近账单
- 账单明细页：表格展示，支持筛选（时间、分类、账户、收/支）
- 报表分析页：四个核心图表（分类占比、月度收支趋势、按账户统计、时间段对比）
- 导入导出页：
  - 导入：上传 CSV/Excel，字段映射预览，确认导入
  - 导出：选择时间范围与字段，下载 CSV/Excel
- 分类管理页：预置分类展示 + 自定义分类新增/编辑/停用
- 账户管理页：账户增删改

### 控件规范（关键字段）

- 账单字段：
  - 必填：金额、收/支、分类、账户、备注
  - 自动填：时间（默认当前）
  - 选填：标签
- 分类预置：餐饮、交通、住房、娱乐、工资
- 自定义分类：支持创建、重命名、停用；不可删除已有账单绑定的分类
- 报表默认时间范围：本月
- 导入导出格式：CSV、Excel

## 用户使用流程

### 流程 A：首次登录并开始记账（手机端）

1. 用户打开 App，进入登录页
2. 选择手机号/微信/邮箱完成登录
3. 首次进入后配置默认账户（至少一个）
4. 点击 `+ 记一笔`
5. 填写金额、收/支、分类、账户、备注（时间默认当前，标签可选）
6. 点击 `保存账单` 完成入账
7. 返回首页查看最新账单与当月摘要

### 流程 B：AI 对话式记账（手机端）

1. 进入 `记一笔` 页面并切换到 `AI 对话记账`
2. 输入自然语言，如“今天午饭35”
3. 系统解析并生成确认卡片（金额、类别、时间等）
4. 用户检查后点击 `确认入账`（或先修改后入账）
5. 若用户修改分类，系统记录偏好用于下次分类推荐

### 流程 C：查账与导出（Web 端）

1. 用户登录 Web 端进入 `账单明细`
2. 按时间范围、分类、账户筛选账单
3. 切换到 `报表分析` 查看四类核心图表
4. 进入 `导入导出`，选择时间范围并导出 CSV/Excel

### 流程 D：导入历史账单（Web 端）

1. 用户进入 `导入导出` 页面
2. 上传 CSV/Excel 文件
3. 系统展示字段映射与预览结果
4. 用户确认后执行导入
5. 导入完成后在账单明细与报表中可见新数据

## 补充说明

| 模块 | 规则 | 说明 |
|------|------|------|
| 认证与同步 | 必须登录后使用 | 不提供匿名本地账本 |
| 登录方式 | 手机号 / 微信 / 邮箱 | 三种并行接入 |
| 手机端技术 | React Native | 与团队 JS/TS 技术栈统一 |
| Web 技术 | React + Next.js | 支持管理端页面组织与渲染 |
| 后端技术 | Go + Gin | 提供认证、账单、分类、报表、导入导出 API |
| AI 记账 | 先确认卡片再入账 | 避免误记账造成数据污染 |
| AI 分类学习 | 用户修正后记忆偏好 | 同类输入优先推荐历史修正结果 |
| 报表核心指标 | 分类占比、月度收支趋势、按账户统计、时间段对比 | MVP 固定四项 |
| 数据导入导出 | CSV、Excel | 导入需校验字段完整性 |

## 应用场景

- 上班族通勤后快速记账：在地铁上说一句“今天午饭35”，确认后直接入账
- 学生党控制生活费：每周在报表页看分类占比，识别餐饮或娱乐超支
- 月末复盘消费结构：在 Web 端查看月度收支趋势与时间段对比，找到异常波动
- 多账户统一管理：把现金、银行卡、支付宝、微信流水集中到一个账本
- 历史数据迁移：从旧表格（Excel/CSV）导入历史账单，再统一导出做备份
